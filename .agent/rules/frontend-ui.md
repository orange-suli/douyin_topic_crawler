# 适用范围: 所有 `*.html`, `*.js`, `*.css` 或前端框架文件

在构建数据看板时，必须注重用户体验和前后端解耦：

## 1. 接口调用与异常处理
* **动态配置:** 严禁在多处散落硬编码的 `http://localhost:8000`。必须在全局或配置模块中定义统一的 `API_BASE_URL`。
* **优雅降级:** 在请求后端接口时（使用 `fetch` 或 `axios`），必须包含完整的 `try...catch` 逻辑。
  * 如果后端报错或无响应，前端必须在 UI 上渲染友好的提示（例如：“数据加载失败，请检查服务状态”），绝不能让页面直接白屏。

## 2. 界面与图表交互
* **响应式设计 (Responsive):** 页面布局必须使用 Flexbox 或 CSS Grid。当浏览器窗口缩放时，图表容器必须自适应伸缩。
* **ECharts 规范:** * 实例化 ECharts 时，必须监听 `window.onresize` 事件以调用 `chart.resize()`。
  * 图表的 Tooltip（悬浮提示）必须开启，确保用户鼠标移入时能看到具体的精确数值（不能只有图形没有数据）。
  * 若某项统计数据为空（如无标签数据），图表区域应显示“暂无数据”的占位符，而不是渲染一个诡异的空坐标系。
